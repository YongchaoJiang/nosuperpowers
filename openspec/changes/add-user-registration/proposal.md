## Why

目前博客只有 `server.js` 里种子生成的单个管理员账号，访客没有办法创建自己的账号。加上邮箱+密码注册后，博客就能识别单个访客身份（比如未来支持作者身份发文、以身份发表评论、个性化等），而不是把所有人都当成匿名访客处理。

## What Changes

- 新增公开的注册表单（`GET /register`）和提交处理路由（`POST /register`），用邮箱和密码创建新用户。
- 校验邮箱格式和唯一性，并在创建账号前校验密码最小长度。
- 用 `bcryptjs`（已是依赖，管理员账号已经在用）对密码做哈希后再存入 `users` 集合。
- 扩展 `users` 集合的 schema（目前只有基于用户名的管理员账号），支持基于邮箱的公开账号，并与管理员账号区分开。
- 复用已有的基于 IP 的内存限流模式（`isRateLimited`/`recordSubmission`，目前只用于评论提交）来限制注册尝试频率，降低滥用/垃圾注册的风险。
- 注册成功后，立即为该访客建立会话（复用已有的 `express-session` 配置），让访客注册后直接处于已登录状态。
- 注册表单在校验失败时（邮箱无效、邮箱重复、密码强度不够）内联展示错误信息，不做整页跳转以免丢失已填内容。
- 把现有的 `/admin/login`（目前只服务唯一的管理员账号）改造成面向所有账号的通用登录页：任何注册用户或管理员都可以用邮箱/用户名+密码登录，登录成功后按角色分别跳转（管理员进后台，普通用户暂时跳转回首页，等后续变更引入用户中心页面后再调整跳转目标）。
- 登录尝试同样复用现有的 IP 限流辅助函数，防止暴力破解密码。

范围之外：密码重置、邮箱验证/确认，以及任何消费账号身份的功能（例如发布文章、以登录用户身份管理评论）。这些属于角色权限体系，交给后续依赖本变更的另一个 change 处理。

## Capabilities

### New Capabilities
- `user-registration`：面向访客的注册流程（表单 + 处理逻辑），允许访客用邮箱和密码创建账号，包含校验、防重复邮箱、密码哈希和频率限制。
- `user-login`：通用登录能力，让注册用户和管理员都能用统一的登录页认证并建立会话，包含凭证校验、失败提示、按角色跳转、登录频率限制。原有的管理员登录行为并入这个能力，不再是 admin 专属的独立逻辑。

### Modified Capabilities
（无——目前没有已有的 spec 描述现状；管理员账号的种子逻辑属于实现细节，不是已文档化的能力）

## Impact

- **代码**：`server.js`（新增注册路由、改造现有 `POST /admin/login` 为通用登录路由、校验逻辑、复用限流辅助函数）、新增 `views/register.ejs` 模板、改造现有 `views/login.ejs` 文案使其不再专属于管理员、可能需要在 `views/partials/` 里新增一个共享的表单错误展示片段。
- **数据**：`blog.json` 的 `users` 集合会在现有管理员记录之外增加基于邮箱的公开用户记录，并新增 `role` 字段区分管理员和公开账号，避免管理后台路由（`requireAuth`）被公开账号意外访问。
- **依赖**：无新增——复用 `package.json` 中已有的 `bcryptjs` 和 `express-session`。
- **会话**：注册和登录都会填充 `req.session`，所以后续依赖 `req.session.userId` 的代码需要同时兼容管理员会话和公开用户会话。
- **后续依赖本变更的工作**：文章归属（authorId）、基于角色的发布/评论审核权限、用户中心页面，属于另一个依赖本变更的独立 change，不在本次范围内。
