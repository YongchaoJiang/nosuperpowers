## ADDED Requirements

### Requirement: 评论提交要求登录
系统 SHALL 只允许已登录用户提交评论，未登录访客提交 `POST /article/:id/comments` 时 SHALL 被拒绝并引导至登录/注册入口。

#### Scenario: 未登录访客尝试评论
- **WHEN** 未建立会话的访客提交 `POST /article/:id/comments`
- **THEN** 系统不创建评论，提示需要登录，并展示登录/注册入口

#### Scenario: 已登录用户提交评论
- **WHEN** 已登录用户提交 `POST /article/:id/comments`，且内容通过其余校验（内容非空、长度限制等既有规则）
- **THEN** 系统创建一条与该用户账号关联的评论

### Requirement: 评论记录关联提交者身份
系统 SHALL 在每条新创建的评论上记录提交者的账号 id（`userId`），不再依赖提交时自由填写的昵称字段来标识评论者。

#### Scenario: 评论写入时记录账号 id
- **WHEN** 已登录用户成功提交一条评论
- **THEN** 该评论记录的 `userId` 等于提交者的账号 id

### Requirement: 公开展示身份来自账号
系统 SHALL 在公开展示评论时，使用提交者账号邮箱 `@` 之前的部分作为显示名称，不展示完整邮箱地址；不再提供自由填写昵称的表单字段。

#### Scenario: 评论公开展示时使用邮箱前缀
- **WHEN** 一条已通过审核的评论被公开展示
- **THEN** 系统展示的评论者名称是其账号邮箱 `@` 之前的部分，而不是完整邮箱或自由填写的文本

#### Scenario: 评论提交表单不再包含昵称/邮箱输入框
- **WHEN** 已登录用户查看评论提交表单
- **THEN** 表单只包含评论内容（及回复目标），不包含昵称或邮箱输入框
